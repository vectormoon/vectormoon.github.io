信号和槽函数

信号和槽函数概论

通过点击按钮使窗口执行操作，比如说：通过点击界面右上角的叉关闭窗口。该操作如如下几个步骤：

  1. 信号发送（右上角的叉）
  2. 发送的具体信号（点击操作）
  3. 信号接收（整个窗口页面）
  4. 信号处理（槽函数：执行关闭窗口功能）

信号槽的优点：松散耦合，发送端和接收端本身没有关联

信号和槽函数的连接使用

使用connect函数将发送段和接收端耦合在一起，可以实现上述的功能。connect函数的定义：

connect函数分别有四个参数：

  1. sender:信号的发送者
  2. signal:发送的信号（传入函数地址）
  3. receiver:信号接收者
  4. method:处理的槽函数（传入函数地址）

signal

其中发送的信号有以下几种信号：

  • clicked函数表示点击操作
  • pressed函数表示按钮按下去
  • released函数表示按钮释放
  • toggled函数表示按钮的状态切换

method

下面实现点击button来关闭窗口的示例：

   QPushButton *btn2 = new QPushButton("btn2", this);
//需求：点击btn2关闭窗口
   connect(btn2, &QPushButton::clicked, this, &myWidget::close);

下课后，老师触发一个信号，下课，学生响应信号，去食堂帮老师打饭

首先需要右键项目选择add new->C++ class创建要实现的类（Teacher类和Student类）。

因为老师发送一个信号，需要在teacher.h中实现signals；学生接受信号并作出相应反应，需要在student.h中实现槽函数

Teacher.h

  • 自定义信号，需要写到signals下；

  • 函数的返回值是void，只需要声明，不需要实现；

  • 函数可以有参数，也可以重载
#ifndef TEARCH_H
#define TEARCH_H

#include <QObject>

class Tearch : public QObject
{
    Q_OBJECT
public:
    explicit Tearch(QObject *parent = nullptr);

signals:
    void hungry();
};

#endif // TEARCH_H

Student.h

  • 早期版本需要写在public slots下，5.0之后可以直接写在public下，或者写成全局函数
  • 返回值是void，需要声明，需要实现
  • 可以有参数，也可以重载
#ifndef STUDENT_H
#define STUDENT_H

#include <QObject>

class Student : public QObject
{
    Q_OBJECT
public:
    explicit Student(QObject *parent = nullptr);
    void treat();
signals:

};

#endif // STUDENT_H

widget.cpp

#include "widget.h"
#include "ui_widget.h"

Widget::Widget(QWidget *parent)
    : QWidget(parent)
    , ui(new Ui::Widget)
{

    ui->setupUi(this);
    //创建老师对象
    this->x = new Tearch(this);
    //创建学生对象
    this->y = new Student(this);
	//连接
    connect(x, &Tearch::hungry, y, &Student::treat);
    //调用下课函数
    classIsOver();
}

void Widget::classIsOver(){
    //下课，触发老师下课信号
    emit x->hungry();
}

Widget::~Widget()
{
    delete ui;
}

拓展

  1. 信号可以连接信号
  2. 一个信号可以连接多个槽函数
  3. 多个信号可以连接一个槽函数
  4. 信号和槽函数的参数必须类型一一对应
  5. 信号和槽的参数个数，信号的参数个数可以多于槽的参数割个数
